Guia funcional · Para times de negócio, produto e pricing

Como o BRE Funciona
Sem código, sem jargão técnico — entendendo o Motor de Regras de Negócio

Este guia explica como o Business Rules Engine (BRE) calcula preços, como você pode ler e solicitar alterações nas regras, e como o sistema garante auditabilidade de cada cálculo. Se você trabalha com pricing, produto ou vendas, este é seu documento. A POC Completa (4ª e última POC) validou os 12 mecanismos de precificação em 18 cenários — incluindo carrinhos de 200 itens com 0 SOQL.

O que é o BRE — em linguagem simples

O BRE é como uma planilha inteligente que vive dentro do Salesforce: você define as regras em tabelas com colunas e linhas, e o sistema automaticamente aplica a regra certa quando um vendedor coloca um produto no carrinho.

🗂 Como era — Apex (código)
  • Regras escritas em código — só desenvolvedor lê e altera
  • Para mudar um preço: abrir chamado → sprint de desenvolvimento → deploy → release
  • Não tem "versão anterior" — rollback exige novo deploy
  • Auditoria: "preciso olhar no log do Salesforce" — nenhum negócio consegue fazer isso
  • A Pricing Matrix tem 4.877 linhas (sandbox) sem contexto — ninguém sabe o que 62% delas faz
✅ Como fica — BRE (declarativo)
  • Regras em tabelas — qualquer pessoa treinada lê e entende
  • Para mudar um preço: abrir a tabela → editar a linha → publicar nova versão (sem deploy)
  • Versionamento automático — a versão anterior fica arquivada, rollback em 1 clique
  • Auditoria nativa: cada step do cálculo exibe valor de entrada, regra aplicada e resultado
  • Tabelas menores e com propósito claro — fácil de entender e manter

Como o preço é calculado — o pipeline

Quando o vendedor adiciona um produto ao carrinho, o BRE executa uma sequência de steps em ordem. Cada step transforma ou enriquece o preço. Pense como uma linha de montagem:

🛒
Produto no Carrinho
Entrada: produto, UF, canal, forma de pgto
📋
Preço Base
Decision Matrix devolve preço tabelado por produto
🗺️
Ajuste Regional
Se existe preço para a cidade, aplica; senão, fallback para UF/nacional
💳
Forma de Pagamento
Ajusta preço para boleto, digital, etc.
🎁
Desconto / Bônus
Aplica promoções e bônus condicionados ao perfil
✅
Preço Final
Retorna ao carrinho com breakdown completo
💡
Cada bloco acima é chamado de Step dentro de um Expression Set. O conjunto de todos os steps é o pipeline de precificação. Você não precisa saber programar para ler, entender ou propor mudanças em cada step.

A Decision Matrix — a tabela de regras

Uma Decision Matrix é como uma tabela de Excel com entradas e saídas. O BRE recebe alguns valores (ex.: produto + modalidade), consulta a tabela e retorna o resultado correto. Veja um exemplo simplificado:

Produto ENTRADA Modalidade ENTRADA Prazo (meses) ENTRADA Canal ENTRADA Preço Base (R$) SAÍDA
Internet Fixa 200MB Mensal * * 79,90
Internet Fixa 200MB Anual 12 * 69,90
Internet Fixa 500MB Mensal * Loja 99,90
Internet Fixa 500MB Mensal * Digital 89,90 ← match!
Internet Fixa 1GB * * * 139,90

O asterisco * significa "qualquer valor" — é uma regra wildcard. A linha em verde mostra o match para o produto "Internet Fixa 500MB" comprado no canal Digital.

📌
O que muda em relação à Pricing Matrix atual: em vez de uma tabela gigante com 4.877 linhas (sandbox) e 29 colunas misturando tudo, o BRE usa tabelas menores e focadas — uma por responsabilidade. A tabela de preço base não mistura informações de promoção ou vigência. A POC Completa provou que 12 mecanismos cabem em 12 matrizes declarativas — e que um carrinho de 200 itens roda com 0 SOQL, eliminando o risco de governador que o motor Apex atual enfrenta a partir de 15 itens.

Como solicitar uma mudança nas regras de preço

Com o BRE em produção, o fluxo de mudança de precificação será assim:

1

Identificar o que mudar

Qual produto? Qual região? Qual condição (canal, modalidade, forma de pagamento)? O time de pricing identifica a Decision Matrix responsável abrindo o catálogo BRE no Salesforce.

2

Criar ou editar a versão

O BRE permite criar uma versão rascunho da Decision Matrix — sem afetar o preço atual em produção. Você edita a tabela diretamente na interface do Salesforce, como uma planilha. Defina a data de início da nova versão.

3

Testar antes de publicar

Com a versão rascunho, é possível simular o cálculo com entradas reais antes de ativar. O resultado mostra o breakdown completo para você conferir se o preço calculado é o esperado.

4

Publicar a nova versão

Após validação, você ativa a nova versão — o sistema passa a usar as novas regras imediatamente. A versão anterior fica arquivada para consulta ou rollback rápido.

5

Monitorar após a publicação

Verifique nos primeiros pedidos após a mudança se o preço calculado está correto. O breakdown de auditoria facilita essa validação sem precisar de suporte técnico.

Glossário — termos que você vai ouvir

📐 Expression Set — o que é?
É o "pipeline completo" de precificação — a sequência ordenada de todos os steps que transformam o produto em preço final. Pense como uma receita: cada step é um ingrediente aplicado em ordem. Pode existir um Expression Set diferente por tipo de produto, canal ou fluxo de venda.
📊 Decision Matrix — o que é?
Uma tabela de regras com colunas de entrada e colunas de saída. Dado um conjunto de condições (produto, canal, UF...), a matriz retorna o valor correspondente. É o "mapa de regras" do BRE.
🔄 Versão / Versionamento — por que importa?
Toda Decision Matrix e todo Expression Set tem um histórico de versões. Se uma promoção vence ou um preço muda, você cria uma nova versão com data de início — sem deletar o histórico. Isso permite auditoria retroativa: "qual era o preço no dia X?" é respondível.
🌎 COALESCE geográfico — o que é?
É a lógica de fallback de preço regional. O BRE tenta encontrar um preço nessa ordem: Node específico → Cidade → DDD → UF → Nacional. Se não existe preço para a cidade, ele usa o da UF. Se não existe preço para a UF, usa o nacional. Isso evita que o vendedor receba erro porque uma localidade não foi cadastrada.
🎁 OTC — o que é?
One-Time Charge — taxa cobrada uma única vez na adesão do produto (ex.: taxa de instalação ou equipamento). É calculada separadamente do preço recorrente mensal e aparece no breakdown do carrinho.
📦 Bundle — o que é no contexto de preço?
Um Bundle é um pacote de produtos (ex.: Internet + TV + Celular) onde o preço final pode depender da combinação. O BRE calcula o preço de cada produto individualmente e pode aplicar um desconto de portfólio por cima — tudo declarativo, sem código.
