POC · Preço Avançado — a prova de que o BRE comporta tudo

A POC do Preço Composto mostrou o núcleo (preço-base + geografia + pagamento + adesão). Esta página vai além: pega os mecanismos que sobravam no motor custom do cliente e prova que todos encaixam nos mesmos blocos do BRE — sem paradigma novo e sem uma linha de código. Um Expression Set (PrecoAvancado) orquestra 6 matrizes e devolve preço mensal, adesão e multa.

Complementa poc-bre-composto.html (núcleo) e visao-negocio-precificacao.html (visão de negócio).

Em uma frase

Cada regra de preço do engine antigo vira uma matriz independente; o Expression Set apenas compõe o resultado com expressões simples (escolher de/por pela jornada, deixar o bundle vencer o base, somar descontos/bônus, expor adesão e multa). Adicionar um mecanismo = adicionar uma matriz, não reabrir código.

1 · Como funciona — visão única

As entradas alimentam as 6 matrizes; o motor compõe e devolve três saídas. A multa aparece à parte (laranja): é informação contratual, não soma no preço.

Mesmo diagrama da página 2 de bre-arquitetura-claro-compactado.drawio (editável no draw.io / diagrams.net).

2 · Os 6 mecanismos — o que cada um resolve

Cada linha é uma Decision Matrix isolada (1 responsabilidade). O ES só decide como combiná-las.

4 Instalação — OTC geográfica AvcInstalacaoMatrix produto × UF* (curinga) Taxa única (one-time). UF específica vence o curinga nacional *. BRE

Por que há desconto/bundle aqui se "promoção fica fora do BRE"?

A POC absorve todos os mecanismos do motor antigo para provar viabilidade e servir de instrumento de decisão — não para fixar tudo como matriz. A régua é a natureza, não o nome: promoção (tem vigência, código, campanha) → Promotions nativas; política/preço estrutural (permanente, sem data de fim) → BRE. E mesmo o que vai para o nativo não dispensa o BRE: a promoção incide sobre o preço que o BRE calcula — as matrizes de preço-base seguem sendo a fonte.

6 · Como estender daqui pra frente

Nova regra de preço
Cria-se uma nova Decision Matrix com a chave certa e um step de lookup no ES. Nada de Apex.
Novo valor real
Para desconto/bônus/instalação reais, basta atualizar a linha da matriz — a estrutura já existe.
Promoção / vigência
Promoções com data de início/fim saem para o motor nativo de Promotions (roadmap), fora da grade de preço.
Lição de processo (registrada no guia)

Uma matriz já ligada a um Expression Set não pode ser recarregada in-place (mesmo com o ES inativo ela segue referenciada). Regra de ouro: carregar todos os dados antes de ligar o ES. Por isso vale superdimensionar a carga inicial.
